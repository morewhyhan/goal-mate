# Frontend Standard

## UI 与设计系统

- 前端样式遵守 `docs/designs/design-system.md`。
- 组件以业务语义命名，避免只按视觉命名。
- 表单必须包含校验、错误提示、加载态和禁用态。
- 关键交互必须可用键盘操作，并保留可访问名称。

## 状态与数据

- 服务端事实由接口返回，不在前端重复推导长期业务状态。
- 页面状态、表单状态、远程数据状态分离管理。
- 失败场景必须有明确反馈，不静默失败。

## E2E 友好

- 关键按钮、输入框、状态区域应具备稳定可访问文本或测试定位属性。
- 不依赖脆弱 CSS 选择器作为唯一自动化测试入口。
