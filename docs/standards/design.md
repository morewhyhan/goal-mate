# 设计规范

## 设计理念

Goal Mate 的设计不是极简主义本身，而是“行动优先的低熵界面”。页面要减少用户判断成本，让用户一眼知道目标、进度、下一步和可操作入口。

## 基本原则

- 一屏先清楚：首屏必须说清楚页面目的和当前最重要信息。
- 信息少但不空：删掉非必要内容，但保留能支撑判断的关键上下文。
- 操作真实：出现的控件必须能配置、切换、编辑或执行。
- 方法论隐藏：OKR、甘特图、日志层级可以存在，但不能让用户像学习项目管理软件一样学习本产品。
- 专业而不玩具：Agent 页面要像工作台，不像演示卡片。

## 页面视觉边界

- Today：主视觉集中在“下一步行动”，热力图作为独立小面板横向展开。
- Goals：使用 OKR + 甘特图/周期视图，强调目标拆解和完成情况，只读。
- Logs：使用接近 VS Code 文件树的层级列表，Markdown 编辑区清晰、直接。
- Agent：左侧历史，中间对话，底部固定输入；只有消息区滚动。
- Settings：左侧分类，右侧配置；每个配置必须说明开启/关闭或填写后的实际影响。

## 视觉语言

- 使用克制的专业工具感：清晰层级、稳定间距、强对齐、少量强调色。
- 不使用大面积无意义渐变、装饰块和花哨动效。
- 卡片可以存在，但不能变成一堆小面板平铺。
- 同一页面只能有一个主标题，不重复出现页面名。
- 标签、胶囊、切换项必须服务于信息分层，而不是装饰。

## 文案原则

- 标题说对象，说明说影响，按钮说动作。
- 少用抽象词，多用用户能直接判断的结果。
- 设置项必须说明“开启后怎样，关闭后怎样”。
- Agent 提示语要强调它能读取目标和日志，并帮助用户梳理和控制系统。

## 交互原则

- 聊天输入框固定可见。
- 列表和编辑区要各自滚动，不能让整个页面失控滚动。
- 热力图默认显示年度，允许切换季度、月度、周度。
- Logs 层级用连线表达父子关系和同级关系，避免一层套一层的笨重卡片。
