# 安全规范

## 鉴权

- 所有用户数据接口必须要求登录态。
- session 是唯一可信身份来源。
- 前端传入的 userId 只能作为非法输入处理，不能用于权限判断。

## 授权

- 读取、更新、删除资源前必须校验资源属于当前用户。
- Agent 只能访问当前用户授权的数据：目标、日志、设置和对话历史。
- 外部机器人接入必须绑定用户身份和 workspace/channel 身份。

## 密钥和模型配置

- DeepSeek、OpenAI 或其他模型 API Key 不得明文返回前端。
- 设置页只能显示脱敏状态，例如 `sk-****abcd`。
- 测试连接接口只返回连接是否可用和必要错误提示。
- 日志中不能打印完整 API Key、Authorization header、cookie。

## 隐私

- 用户日志可能包含高度私密的生活、身体、学习和项目数据，默认不公开、不共享。
- 导出 Markdown 是用户主动行为。
- 删除账号必须明确处理目标、日志、对话和设置数据。

## Agent 安全边界

- Agent 可以建议和整理，但高风险操作必须经过用户确认。
- Agent 不能静默删除大量日志或目标。
- Agent 使用外部渠道发送提醒时，必须遵守用户设置的时间、频率和渠道权限。

## 前端安全

- Markdown 渲染必须防 XSS。
- 用户输入不能直接作为 HTML 注入。
- 文件路径和日志名称必须校验，禁止路径穿越。
