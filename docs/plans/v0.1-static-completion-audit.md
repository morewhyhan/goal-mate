## 4. 风险清单

| 风险 | 说明 | 处理方式 |
| --- | --- | --- |
| Prisma 枚举和 Hono 输入值不一致 | API 当前使用 Prisma 枚举大写值，前端/测试可能传小写 | typecheck/API 测试后修正映射 |
| Hono RPC 类型路径可能不匹配 | hooks 使用 Hono client 路径推导，需 typecheck 证明 | 运行 `pnpm typecheck` 后修复 |
| SQLite 迁移可能受旧 Ignite 迁移影响 | 旧迁移来自 task 模板 | 优先本地 reset，或重新整理迁移 |
| Better Auth 与重写 user 关系可能有兼容问题 | 保留了 Better Auth 模型，但未运行 | generate/migrate 后验证登录 |
| 页面 hooks 在未登录时会收到 401 | 页面不再展示 demo fallback，未登录/无数据时只能显示真实空状态 | 验收时决定是否加更明确的登录提示 |
| Agent 真实模型能力依赖配置 | DeepSeek 配置已接入，但当前审计未运行真实模型调用 | 运行模型连接测试和 Agent Loop 验收 |

## 5. 下一步唯一有效动作

进入 `PLAN-025` 实际验收。需要用户明确授权运行以下命令之一组：

```bash
cd src
pnpm db:generate
pnpm db:reset
pnpm db:seed
pnpm typecheck
pnpm verify:v01:api
```

如果不允许 reset，则使用：

```bash
cd src
pnpm db:generate
pnpm db:migrate
pnpm db:seed
pnpm typecheck
pnpm verify:v01:api
```
