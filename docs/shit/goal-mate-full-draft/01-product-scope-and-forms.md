# F0 产品边界与形态规格

## 1. 业务背景

Goal Mate 不做单一场景陪跑产品，也不做传统 Todo 或企业 OKR。它是一个目标推进系统，底层用统一的目标推进引擎，上层适配不同目标场景和部署形态。

## 2. 产品定位

| 不是 | 是 |
| --- | --- |
| 普通待办工具 | 目标推理与推进系统 |
| 传统 OKR 管理软件 | 隐式 OKR 秘书 |
| 简单打卡提醒 | 行为诊断与路径调整系统 |
| 全自动替用户决策 | 需要确认的主动秘书 |
| 三个垂直产品 | 一个通用产品，多个目标场景 |

## 3. 用户角色

| 角色 | 说明 | 主要权限 |
| --- | --- | --- |
| Personal User | 普通个人用户 | 创建目标、对话、执行、复盘、配置提醒 |
| Power User | 技术/隐私用户 | Personal User 权限 + 自部署、模型配置、MCP、导出 |
| Admin | 云端运营管理员 | 用户支持、系统配置、模板管理、审计，不可读取用户敏感内容明文 |
| Integration Actor | 外部工具或 MCP Client | 根据授权读取或写入目标上下文 |

## 4. 完整版能力边界

### 4.1 必须做

| 能力 | 优先级 |
| --- | --- |
| 多目标管理，但要求当前主目标 | P0 |
| 目标澄清、目标推理卡、隐式 OKR | P0 |
| 关键条件倒推、阶段节点、今日行动 | P0 |
| 每日 Check-in、未完成诊断、路径调整 | P0 |
| 日/周/月/目标周期复盘 | P0 |
| 用户画像、行为模式、提醒偏好 | P0 |
| Web Console 与 Cloud API | P0 |
| 日历、邮箱、消息等外部上下文读取 | P1 |
| Self-hosted Agent 与 MCP Server | P1 |
| 移动 App / 小程序 | P1 |
| 多人协作与组织版 | P2 |

### 4.2 明确不做或受限做

| 能力 | 规则 |
| --- | --- |
| 医疗、法律、金融专业建议 | 只做目标推进，不做专业结论 |
| 自动对外承诺 | 必须用户确认 |
| 自动花钱、签约、发邮件、删除数据 | 默认禁止，P2 也必须强确认 |
| 评价用户人格或意志力 | 禁止 |
| 无限目标并行 | 限制 active 目标数量，必须选择 current_focus_goal |

## 5. 产品形态

### 5.1 Cloud App

面向普通用户，提供：

```text
账号登录
Web / App / 小程序入口
云端目标数据
跨设备同步
主动提醒
模型服务托管
基础数据导出
```

### 5.2 Self-hosted Agent

面向程序员、隐私用户，提供：

```text
本地 Web UI
Local API
MCP Server
SQLite / 本地数据库
模型 Provider 配置
Scheduler
连接器配置
日志、备份、导出
```

### 5.3 Goal Engine Core

所有形态共用，负责：

```text
目标理解
目标推理
条件倒推
计划生成
行动选择
诊断调整
复盘洞察
结构化 AI 输出校验
```

## 6. 业务规则

| 编号 | 规则 | 优先级 |
| --- | --- | --- |
| F0-R1 | Core 不得依赖 Cloud App 或 Self-hosted Agent 的 UI 实现 | P0 |
| F0-R2 | Cloud 和 Self-hosted 必须共用同一套核心数据模型 | P0 |
| F0-R3 | 用户可以创建多个目标，但同一时间只能有一个 current_focus_goal | P0 |
| F0-R4 | 外部工具读取或写入目标数据必须有授权记录 | P0 |
| F0-R5 | 所有自动化外部动作必须区分 read、draft、execute 三个权限层级 | P1 |
| F0-R6 | 自部署用户必须能导出全部本地数据 | P1 |
| F0-R7 | 云端管理员不得直接查看用户完整私密对话明文，除非用户授权支持 | P1 |

## 7. 验收标准

| 编号 | Given | When | Then |
| --- | --- | --- | --- |
| AC-F0-1 | 用户创建多个目标 | 打开今日页 | 系统展示 current_focus_goal，并解释优先级 |
| AC-F0-2 | 自部署服务启动 | 用户打开本地地址 | 可访问 Web UI，并能看到服务状态 |
| AC-F0-3 | 外部工具请求执行动作 | 未获得 execute 权限 | 系统拒绝并要求用户确认 |
| AC-F0-4 | 云端用户请求导出 | 导出完成 | 用户获得目标、行动、复盘、设置等完整数据包 |
