# F14 用户行为因子与干预模型规格

状态：后续增量需求，不进入 v0.1 实现范围。  
当前阶段：用普通 LLM 基于目标、日志、反馈和元认知记录模拟该能力。  

## 1. 模块定位

用户行为因子与干预模型是 Goal Mate 后续的个性化控制层。

它要解决的问题不是“让 AI 更会聊天”，而是：

```text
如何基于用户过去的日志、反馈和行动结果，学习这个用户在什么条件下会行动、会拖延、会抗拒、会失控，并据此生成更合理的下一次干预。
```

类比量化交易：

```text
量化交易用历史 K 线、成交量、波动、回撤等数据训练策略；
Goal Mate 用历史日志、目标状态、行动反馈、偏差记录和干预结果训练用户行为干预策略。
```

但是第一版不能直接做黑箱训练。第一阶段必须先形成：

```text
日志数据 -> 因子抽取 -> 可解释评分 -> 干预策略 -> 反馈标签 -> 因子更新
```

## 2. 与 v0.1 的关系

v0.1 不训练专门模型。

v0.1 只需要做到：

| 能力 | v0.1 处理方式 |
| --- | --- |
| 因子识别 | LLM 从日志、反馈、目标状态中临时推理 |
| 干预选择 | Intervention Planner 基于 LLM 和规则生成 |
| 用户模型更新 | Meta-Cognition 记录可验证假设 |
| AI 自我更新 | AiSelfReflection / AiSelfOptimizationUpdate 记录下一次推理规则 |
| 训练数据沉淀 | Logs、ControlLoopEpisode、Checkin、Review、Meta-Cognition 保留足够结构 |

后续版本才可以在这些数据之上训练个性化模型。

因此，本规格的意义是：

```text
当前不实现训练，但现在的日志、元认知和控制回合必须为未来训练留下正确数据。
```

## 3. 数据来源

未来行为因子模型可以读取以下数据。

| 数据 | 作用 |
| --- | --- |
| Goal / KeyResult | 用户想达到什么结果，成功标准是什么 |
| StagePlan / DailyAction | 系统给出的阶段计划和今日行动 |
| Checkin | 用户完成、部分完成、未完成或无响应 |
| Markdown Logs | 用户每天事实、偏差、原因、控制策略和自由记录 |
| 用户已有日记/日志 | 如果用户有长期记日记习惯，可作为冷启动参考 |
| ControlLoopEpisode | 每次干预、反馈、诊断、状态变化的闭环记录 |
| InterventionDecision | Agent 当时为什么这样问、这样提醒或这样降难度 |
| Meta-Cognition | 对用户模型和 AI 自身策略的持续更新 |
| Scheduler / Channel Event | 提醒时间、渠道、是否触达、是否回复 |

用户已有日记、日志或 Markdown 资料只能作为参考输入，不能直接当作事实结论。系统必须区分：

```text
用户原始叙述
AI 解释
可验证行为事实
后续反馈支持或否证的判断
```

## 4. 因子体系

行为因子不是人格标签，而是用于预测和干预的可变状态。

| 因子 | 含义 | 可观察信号 |
| --- | --- | --- |
| 目标真实度 | 用户是否真的想要该结果，而不是外部期待 | 长期回避、语气抗拒、完成后无正反馈 |
| 行动仓位 | 今日任务是否过大，是否一次失败就击穿系统连续性 | 连续未完成、用户说太难、开始前就放弃 |
| 启动阻力 | 从无行动到开始行动的摩擦 | “晚点”“不想动”“打开都烦” |
| 当前能量 | 用户当天身体、情绪、时间承载能力 | 疲惫、生病、加班、情绪波动 |
| 环境支持度 | 当下环境是否允许行动发生 | 工具不可用、地点不合适、被打断 |
| 提醒有效性 | 当前提醒时间、渠道、语气是否有效 | 是否回复、是否行动、是否反感 |
| 反馈质量 | 用户反馈是否足以支持判断 | 只回“没做”、沉默、能否说明原因 |
| 连续拖延 | 同一目标或行动连续偏离次数 | 多日未完成、重复推迟 |
| 多目标负载 | 用户同时推进目标的总压力 | 目标过多、计划挤压、疲于切换 |
| 目标冲突度 | 多个目标之间是否互相抢资源 | 早睡和夜间写作冲突、减肥和外卖冲突 |
| 风险窗口 | 用户最容易失控的时间、地点、情境 | 晚上点外卖、睡前刷手机 |
| 干预响应 | 用户对某类干预是否更容易行动 | 接受最小动作，反感强催 |
| AI 误判风险 | Agent 上一次判断是否被后续反馈否证 | 误判为懒、误判为提示不足、计划过大 |

所有因子都必须是可更新的假设，不得写成固定人格判断。

不允许：

```text
用户懒。
用户自控力差。
用户就是不行。
```

允许：

```text
在最近 3 次英语任务中，用户对“打开教材并背诵”的启动阻力较高；对“走路时听 3 分钟音频”的接受度更高。该判断仍需后续验证。
```

## 5. 模型输出

未来模型每次决策前应输出可解释结果。

| 输出 | 说明 |
| --- | --- |
| `factor_snapshot` | 当前关键因子及其证据 |
| `completion_probability` | 当前行动完成概率的估计 |
| `risk_probability` | 当前计划失控或偏离的风险估计 |
| `load_assessment` | 多目标总负载是否过高 |
| `intervention_recommendation` | 建议缩小、重排、暂停、追问、提醒或复盘 |
| `reasoning_summary` | 为什么这样判断 |
| `policy_delta` | 下一次干预策略如何变化 |
| `verification_signal` | 用什么后续反馈验证这次判断 |

模型输出必须能被审计。不能只输出“推荐这样做”，必须说明：

```text
基于哪些日志
识别了哪些因子
为什么选择这个干预
下次如何证明它有效或无效
```

## 6. 日志如何变成训练数据

行动控制日志是未来训练的核心数据资产。

一份合格日志不仅记录今天做了什么，还要记录：

```text
计划是什么
实际发生什么
偏差在哪里
用户为什么没做
AI 当时如何判断
AI 的判断后来是否被支持
下一次干预如何改变
用户是否因此更容易行动
```

日志中的结构化区块应支持未来抽取训练样本：

| 日志区块 | 可形成的训练信号 |
| --- | --- |
| 今日计划 | Agent 原始输入和预期行动 |
| 实际完成 | 标签：done / partial / not_done / no_response |
| 未完成原因 | 偏差类型标签 |
| 风险点 | 风险因子 |
| 控制策略 | 干预动作 |
| 明日调整 | 策略变化 |
| 用户模型更新 | 因子更新 |
| Agent 策略更新 | AI 自我优化标签 |

## 7. 训练路线

该能力应按阶段推进。

| 阶段 | 能力 | 说明 |
| --- | --- | --- |
| Phase 0 | LLM 模拟 | 当前阶段。LLM 读取日志和反馈，临时推理因子和干预 |
| Phase 1 | 可解释因子评分 | 将关键因子结构化记录，形成简单评分和规则 |
| Phase 2 | 个体行为模型 | 对单个用户形成个性化完成概率和风险估计 |
| Phase 3 | 跨用户策略模型 | 在隐私合规前提下学习通用干预规律 |
| Phase 4 | 自动策略评估 | 对干预策略做 A/B、反事实和长期效果评估 |

不能跳过 Phase 1。没有可解释因子和清楚标签，直接训练模型会学到伪相关。

## 8. 可解释性与安全边界

该模型干预的是人的行动，因此必须保持可解释和可控。

| 边界 | 要求 |
| --- | --- |
| 用户可见 | 关键判断应能在日志中解释 |
| 用户可改 | 用户可以编辑日志、纠正 AI 判断 |
| 高风险确认 | 暂停目标、重大重排、删除记录等必须确认 |
| 不做人格评判 | 只描述状态、证据和假设 |
| 不做隐性操控 | 不使用羞辱、恐惧、强迫或欺骗性话术 |
| 不黑箱决策 | 重要干预必须能说明依据 |

Goal Mate 可以把用户当作一个可观察、可反馈、可调参的行动系统，但不能剥夺用户最终选择权。

## 9. 后续验收标准

| 编号 | Given | When | Then |
| --- | --- | --- | --- |
| AC-F14-1 | 用户有连续多日日志 | Agent 生成下一次干预 | 必须引用至少一条历史日志或反馈作为因子证据 |
| AC-F14-2 | 用户已有个人日记或旧日志 | 系统导入作为冷启动参考 | 必须标记为用户原始材料，不得直接当作事实结论 |
| AC-F14-3 | 用户连续拖延同一目标 | 模型评估状态 | 必须输出启动阻力、行动仓位或目标真实度等可解释因子 |
| AC-F14-4 | 干预后用户完成率变化 | 系统复盘 | 必须评估上一次干预策略是 supported / contradicted / inconclusive |
| AC-F14-5 | 模型建议调整计划 | 写入日志 | 日志必须说明调整依据、预期效果和下一次验证信号 |
| AC-F14-6 | 数据不足 | Agent 决策 | 不得伪装成训练模型结论，只能用 LLM 推理或规则，并标注低置信 |

## 10. 一句话

```text
Goal Mate 未来要训练的不是一个会说好听话的模型，而是一个基于日志反馈不断校准的用户行为干预模型。
```
