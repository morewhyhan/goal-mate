# F13 登录注册与会话规格

## 1. 模块定位

认证不是目标推进功能本身，但它是所有个人目标、日志、Agent 记忆、模型配置和提醒配置的边界。

一句话目的：

```text
用户能创建账户或登录，并且只能进入自己的 Goal Mate 工作区。
```

## 2. 充分必要条件

| 条件 | 必要性 |
| --- | --- |
| 可见入口 | 用户首次打开产品时必须知道从哪里登录或注册 |
| 邮箱密码注册 | v0.1 最小账号体系，不依赖第三方平台 |
| 邮箱密码登录 | 老用户必须能回到自己的目标、日志和 Agent 记忆 |
| 会话门禁 | 未登录不能看到 Dashboard 私有页面 |
| 业务 API 鉴权 | 所有用户私有资源必须从 session 推导 userId |
| 模型密钥隔离 | 每个用户必须配置和使用自己的模型 API Key |
| 退出登录 | 用户必须能结束当前设备会话 |
| 无假按钮 | 未实现的第三方登录、忘记密码、手机号登录不展示为可用能力 |

缺少任一项，用户都会出现无法开始、误入空页面、数据串号或以为某功能可用但实际不可用的问题。

## 3. 用户流程

```text
首次打开
-> 看到 Login
-> 选择登录或注册
-> 提交邮箱和密码
-> 进入 Today
-> 使用 Dashboard 五页
-> 可从侧边栏退出登录
```

已登录用户打开 Login 时，系统应直接进入 Today。

未登录用户访问 `/dashboard/*` 时，系统应跳转到 Login，不展示目标、日志、Agent 历史、设置或任何私有数据。

## 4. v0.1 不做

```text
第三方 OAuth 登录
手机号登录
邮箱验证码
找回密码 UI
多租户团队账号
账号注销完整流程
```

这些可以后续扩展，但不能在 v0.1 页面上表现成已可用。

## 5. 页面要求

| 页面 | 要求 |
| --- | --- |
| Login | 登录/注册二选一，字段少，提交动作明确 |
| Dashboard | 登录态确认完成前不渲染私有页面 |
| Sidebar | 显示当前用户，并提供退出登录 |

Login 页面不承担产品说明书职责。它只需要让用户清楚知道：

```text
我输入账号密码，就可以进入自己的目标推进工作区。
```

## 6. 数据与权限规则

| 编号 | 规则 |
| --- | --- |
| AUTH-R1 | 前端不能传入 userId 决定数据归属 |
| AUTH-R2 | 后端只能从 session 获取当前用户 |
| AUTH-R3 | 未登录访问私有 API 必须返回 UNAUTHORIZED |
| AUTH-R4 | 退出登录后 Dashboard 不再展示私有数据 |
| AUTH-R5 | 注册成功后必须能进入当前用户工作区 |
| AUTH-R6 | 当前用户的 ModelConfig、API Key、Agent 调用和数据导出必须按 userId 隔离 |
| AUTH-R7 | 模型 API Key 不能作为服务器全局默认值被所有用户共用 |

## 7. 验收标准

| 编号 | 标准 |
| --- | --- |
| AUTH-AC-1 | `/` 进入登录入口或等价的登录引导 |
| AUTH-AC-2 | 用户能用邮箱密码注册新账户 |
| AUTH-AC-3 | 用户能用邮箱密码登录已有账户 |
| AUTH-AC-4 | 未登录访问 `/dashboard/today` 会被引导到登录页 |
| AUTH-AC-5 | 登录后进入 `/dashboard/today` |
| AUTH-AC-6 | Dashboard 侧边栏能看到当前用户和退出入口 |
| AUTH-AC-7 | 页面没有第三方登录、忘记密码等未实现假入口 |
| AUTH-AC-8 | 两个用户分别配置模型 API Key 后，互相不能读取、导出或使用对方密钥 |
