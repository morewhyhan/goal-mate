# F2 页面规格

## 1. 页面总原则

页面不是展示方法论的地方，而是让用户一眼知道当前状态和下一步。复杂推理可以由 Agent 解释，普通页面只呈现必要结果。

## 2. Login

一句话目的：让用户登录或注册，并进入自己的 Goal Mate 工作区。

必须展示：

| 元素 | 说明 |
| --- | --- |
| 登录/注册切换 | 用户能明确选择已有账号或创建账号 |
| 邮箱 | v0.1 的账号标识 |
| 密码 | 最小认证凭据 |
| 提交按钮 | 登录或创建账户，动作必须真实可用 |
| 产品一句话 | 简短说明这是目标推进工作区，不写长说明 |

不展示：

```text
未实现的第三方登录
未实现的忘记密码
复杂营销落地页
登录后仍可见的无关说明
```

## 3. Today

一句话目的：告诉用户今天下一步做什么。

必须展示：

| 元素 | 说明 |
| --- | --- |
| 当前主目标 | 今天服务哪个目标 |
| 今日行动 | 页面主视觉，只能有一个主行动 |
| 绑定条件 | 这一步补齐哪个关键条件 |
| 完成标准 | 用户怎么判断做完 |
| 最小启动 | 状态差时也能开始 |
| 快捷反馈 | 完成、部分完成、没做、改小 |
| Momentum 热力图 | 独立小面板，默认年度，支持季度/月度/周度 |

不展示：

```text
复杂任务列表
全部 OKR 解释
长篇目标原因
大量统计图
```

## 4. Goals

一句话目的：只读展示目标如何被拆解，以及推进到哪里了。

必须展示：

| 元素 | 说明 |
| --- | --- |
| Objective | 用户真正要达成的方向 |
| Key Results | 证明目标达成的关键结果，一个 O 可以有多个 KR |
| 必要条件 | 当前目标成立所需条件 |
| 当前缺口 | 当前最该补齐的一项或一组条件 |
| 阶段计划 | 目标周期中的阶段节点 |
| 进度 | KR、条件、阶段的完成情况 |
| Gantt/周期视图 | 用项目周期方式展示时间和节点 |

页面规则：

| 规则 | 说明 |
| --- | --- |
| 只读 | Goals 不承担编辑和推进操作 |
| 不展示“为什么这么定” | 如果用户想问原因，去 Agent 问 |
| 不堆解释文字 | 用结构、进度和时间线表达 |
| KR 必须充分必要 | 不限制数量，但每条都要必要且准确 |

## 5. Logs

一句话目的：用 Markdown 记录目标如何被推进，并允许用户直接编辑。

必须展示：

| 元素 | 说明 |
| --- | --- |
| 文件树 | 年/季/月/周/日层级，类似 VS Code 文件列表 |
| 层级连线 | 同级和父子节点用简单竖线表达 |
| Markdown 编辑器 | 用户可以直接编辑当前日志 |
| 自动引用 | 当前目标、KR、今日行动、复盘内容可自动写入 Markdown |
| 保存状态 | 告诉用户是否已保存 |

不做：

```text
知识卡片
MOC
灵感收集箱
复杂标签体系
```

## 6. Agent

一句话目的：用户通过对话让 AI 理解、规划、诊断、复盘和控制系统。

必须展示：

| 元素 | 说明 |
| --- | --- |
| 对话历史 | 用户可以回到不同主题的连续对话 |
| 消息区 | 常规来回对话式体验 |
| 固定输入框 | 输入框始终可见，不能靠滚动寻找 |
| 上下文提示 | 简短显示当前 Agent 能读取目标和日志 |
| 结构化结果 | 关键输出能转成目标、行动、日志或设置草案 |

不展示：

```text
无信息量的说明面板
固定右侧长文案
假审批按钮
玩具式聊天卡片
```

## 7. Settings

一句话目的：配置系统如何理解、提醒、记录、调用模型和保护数据。

必须包含分类：

| 分类 | 必要配置 |
| --- | --- |
| General | 语言、时区、默认周期起点 |
| Goals | 主目标策略、active 目标上限、复盘节奏 |
| Logs | Markdown 根目录、命名规则、自动写入内容 |
| Today | 今日行动生成时间、低精力模式、热力图统计口径 |
| Agent | 对话记忆、可读取范围、结构化输出确认 |
| Models | Provider、模型、API Base、API Key、默认用途、测试连接 |
| Notifications | 提醒时间、渠道、频率、免打扰 |
| Integrations | 微信/飞书/Telegram/日历等后续接入口 |
| Data & Privacy | 导出、备份、删除、隐私边界 |

设置规则：

| 规则 | 说明 |
| --- | --- |
| 不能只有开关 | 模型、日志、提醒等需要具体配置 |
| 必须说明影响 | 开启/关闭或填写后会发生什么 |
| 不允许超出边界 | 内容必须在面板内可读可操作 |
| 不放假按钮 | 没实现的功能要标明后续或禁用原因 |

## 8. 验收标准

| 编号 | 页面 | 标准 |
| --- | --- | --- |
| AC-F2-0 | Login | 用户能注册、登录，未登录不能进入 Dashboard |
| AC-F2-1 | Today | 首屏能看到唯一下一步、完成标准和反馈入口 |
| AC-F2-2 | Goals | 能看到 O、多条 KR、条件、阶段和进度，且没有编辑操作 |
| AC-F2-3 | Logs | 文件树是层级结构，不是年度/季度/月度并列平铺 |
| AC-F2-4 | Agent | 输入框固定可见，只有消息区滚动 |
| AC-F2-5 | Settings | 模型配置包含 B.AI、API Key、API Base、默认模型和测试连接 |
