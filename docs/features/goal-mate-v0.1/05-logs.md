# F4 Logs 规格

## 1. 模块定位

Logs 是 Goal Mate 的推进记录层。它借鉴 Obsidian 式 Markdown 和双向引用思想，但目的不是知识管理，而是沉淀行动推进过程。

## 2. 核心原则

```text
记录服务于推进，不服务于收藏。
```

因此不做 MOC、灵感箱、知识卡片、复杂标签系统。用户如果有灵感，应该马上和 Agent 对话，让它变成目标、条件、行动或日志，而不是单独收藏。

## 3. 层级结构

日志按时间层级组织：

```text
Logs
  2026
    Q3
      2026-07
        W27
          2026-07-01.md
```

左侧展示要像文件树：

| 层级 | 示例 | 说明 |
| --- | --- | --- |
| Year | 2026 | 年度目标和年报 |
| Quarter | Q3 | 季度目标和季报 |
| Month | 2026-07 | 月度推进和月报 |
| Week | W27 | 周计划和周报 |
| Day | 2026-07-01.md | 日行动、反馈和日记 |

## 4. Markdown 内容

### 4.1 日志模板

```md
# 2026-07-01

## 今日主目标

- 目标：
- 当前 KR：
- 当前关键条件：

## 今日行动

- 行动：
- 完成标准：
- 最小启动：

## 执行反馈

- 结果：done / partial / not_done
- 原因：
- 调整：

## 自由记录


```

### 4.2 周志模板

```md
# 2026 W27

## 本周目标

## 本周补齐的条件

## KR 变化

## 未完成原因

## 下周重点

```

### 4.3 月志/季志/年志

月志、季志、年志不是并列按钮，而是文件树中的父级节点。它们汇总下级日志，保留 Markdown 可编辑性。

## 5. 自动引用

系统可以自动把结构化内容写入 Markdown：

| 来源 | 写入内容 |
| --- | --- |
| Goal | 目标标题、Objective、KR |
| DailyAction | 今日行动、完成标准、最小启动 |
| Checkin | 完成情况、未完成原因 |
| Diagnosis | 诊断类别、调整建议 |
| Review | 周/月/年复盘摘要 |

## 6. 编辑规则

| 规则 | 说明 |
| --- | --- |
| 用户可直接编辑 Markdown | 用户拥有最终文本控制权 |
| 空日志可直接开始 | 如果还没有日志，用户可以从 Logs 页面创建今日日志 |
| 自动写入不能覆盖用户手写内容 | 使用区块或追加方式 |
| 系统要保留保存状态 | 例如已保存、保存中、冲突 |
| 删除日志要确认 | 防止误删长期记录 |

## 7. 验收标准

| 编号 | Given | When | Then |
| --- | --- | --- | --- |
| AC-F4-1 | 用户打开 Logs | 有多个年份和季度 | 左侧显示层级树和连线，而不是平铺卡片 |
| AC-F4-2 | 用户点击某日日志 | 页面加载 | 右侧显示 Markdown 编辑器 |
| AC-F4-3 | 用户完成 Today 行动 | 系统保存 | 当日日志追加行动和反馈 |
| AC-F4-4 | 周期结束 | 系统生成 | 周志/月志能汇总下级日志内容 |
