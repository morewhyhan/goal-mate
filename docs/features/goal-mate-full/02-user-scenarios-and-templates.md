# F1 用户场景与目标模板规格

## 1. 模块定位

本模块定义目标用户、目标场景、场景模板和场景化提问策略。模板不是把产品拆成多个版本，而是让同一套目标推进引擎用不同语言理解不同目标。

## 2. 用户画像

| 用户群体 | 典型目标 | 关键需求 |
| --- | --- | --- |
| 学生/备考 | 考研、考公、留学、期末、证书 | 学习节奏、薄弱诊断、长期坚持 |
| 求职/转型 | 简历、作品集、面试、转岗 | 目标岗位、能力缺口、投递节奏 |
| 创业/副业 | MVP、获客、收入、融资 | 关键验证、客户反馈、交付节奏 |
| 内容/作品 | 公众号、视频、作品集、课程 | 选题、产出节奏、发布复盘 |
| 技能成长 | 编程、英语、设计、销售 | 练习路径、反馈、阶段成果 |
| 健康/习惯 | 运动、睡眠、饮食 | 行为发生率、提示时机、低门槛 |
| 自定义 | 用户自定义 | AI 动态澄清场景和条件 |

## 3. 目标模板字段

| 字段名 | 类型 | 必填 | 默认值 | 描述 |
| --- | --- | --- | --- | --- |
| id | string | 是 | 系统生成 | 模板 ID |
| scenario_type | enum | 是 | custom | 场景类型 |
| name | string | 是 | 无 | 模板名称 |
| opening_questions | string[] | 是 | [] | 初始澄清问题候选 |
| success_signal_examples | string[] | 是 | [] | 成功标准示例 |
| common_conditions | object[] | 是 | [] | 常见关键条件 |
| common_blockers | object[] | 是 | [] | 常见阻塞 |
| default_cadence | object | 是 | {} | 推荐提醒和复盘节奏 |
| tone_rules | string[] | 否 | [] | 场景化表达规则 |
| risk_notes | string[] | 否 | [] | 风险提示 |

## 4. 场景模板示例

### 4.1 学习/考试

```yaml
scenario_type: learning_exam
opening_questions:
  - "如果 30 天后这个学习目标真的推进了，最明显的变化是什么？"
  - "现在最不清楚的是目标、当前水平、资料，还是学习节奏？"
common_conditions:
  - "目标考试或科目范围明确"
  - "当前水平被诊断"
  - "每日学习动作足够小"
  - "薄弱模块能被持续复盘"
common_blockers:
  - category: ability
    text: "任务太大或太抽象"
  - category: prompt
    text: "提醒时间不在可学习窗口"
```

### 4.2 创业/副业

```yaml
scenario_type: business_side_project
opening_questions:
  - "这个目标第一阶段要证明客户、收入、产品上线，还是交付能力？"
  - "现在最关键的未知数是什么？"
common_conditions:
  - "目标客户明确"
  - "价值假设可验证"
  - "第一版交付物可做出"
  - "反馈渠道存在"
common_blockers:
  - category: path
    text: "做了很多功能但没有验证关键假设"
```

## 5. 业务规则

| 编号 | 规则 | 优先级 |
| --- | --- | --- |
| F1-R1 | 用户创建目标时可以选择场景，也可以让 AI 推断场景 | P0 |
| F1-R2 | 场景模板只能影响提问、示例和默认条件，不能改变核心流程 | P0 |
| F1-R3 | AI 推断场景的置信度低于 0.6 时，必须让用户确认 | P0 |
| F1-R4 | 每个场景至少要有 3 个 opening_questions 和 5 个 common_conditions | P0 |
| F1-R5 | 用户可把目标切换为其它场景，切换后需重新生成目标推理卡 | P1 |
| F1-R6 | Power User 可自定义模板 | P1 |
| F1-R7 | 模板市场和共享模板属于 P2 | P2 |

## 6. 验收标准

| 编号 | Given | When | Then |
| --- | --- | --- | --- |
| AC-F1-1 | 用户输入“我想准备考研” | AI 推断场景 | scenario_type 为 learning_exam，且提出学习相关澄清问题 |
| AC-F1-2 | 用户输入模糊目标 | AI 置信度低 | 系统要求用户选择或确认目标场景 |
| AC-F1-3 | 用户切换场景 | 保存更改 | 原目标推理卡标记 stale，并提示重新生成 |
| AC-F1-4 | Power User 创建模板 | 字段完整 | 模板可用于新目标创建 |
