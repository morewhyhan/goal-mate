# 单一对话入口与主动推进助手

状态：v0.1 核心体验增量
日期：2026-07-24

## 1. 当前行为

Goal Mate 已有 Web Agent、Today、Goals、Logs、Settings、QQ Worker 和 Scheduler，但用户仍可能需要：

- 在多个页面理解并管理目标结构。
- 在 Web 与 QQ 面对不同的对话逻辑。
- 提交反馈后自己寻找诊断结果和下一步。
- 手工配置多条固定提醒，或在未明确同意时被默认提醒。

这不符合“把中间组织过程托付给 AI”的产品目标。

## 2. 目标行为

Goal Mate 对用户只提供一个逻辑入口：

```text
用户说明目的或现实变化
  -> Agent 理解并维护内部目标状态
  -> 必要时只追问一个关键信息
  -> 用户确认重大目标理解
  -> Agent 给出当前唯一行动
  -> 用户执行后在同一入口反馈
  -> Agent 把反馈变成证据、诊断和状态更新
  -> Agent 返还调整后的下一行动
```

QQ 与 Web 可以是同一入口的不同窗口，但必须使用同一份 Agent 身份、Prompt、权限、记忆、工具运行时和当前状态。

Goals、Logs、Today 继续存在时，只作为按需查看和纠错界面，不要求用户维护内部结构。

## 3. 用户只需要做什么

- 说明想达到的现实结果。
- 回答 Agent 当前唯一必要的问题。
- 执行当前行动。
- 用自然语言反馈完成、部分完成、没做、做不动、情况变化、暂停或放弃。
- 对改变最终目的、成功标准、主目标和高风险外部动作进行确认。

用户不需要手工维护 KR、必要条件、阶段、日志、诊断类别或提醒规则。

## 4. Agent 必须自动做什么

- 把自然语言目的形成可版本化的目标草稿。
- 在确认前用人话展示目标、成功证据、当前缺口、阶段和第一行动。
- 将反馈关联到正确的目标和行动。
- 记录事实、证据、Check-in、Diagnosis、Log 和策略变化。
- 根据现实变化调整行动大小、执行顺序、风险预案和提示时机。
- 每次反馈后明确告诉用户：系统如何理解、改变了什么、下一步是什么。
- 用户询问时解释当前目标、下一步和调整依据。
- 不暴露内部工具名、枚举、Prompt、JSON 或诊断标签。

## 5. 主动干预

用户可以只授予一次“允许 AI 主动联系”的总权限，不需要逐条设置所有提醒。

总权限开启后，Scheduler 只生成候选窗口，Contact Policy 决定本次：

```text
send | skip | defer
```

决策至少考虑：

- 当前行动的计划时间和风险窗口。
- 用户最近一次对话或反馈。
- 今日是否已经完成。
- 是否已有一条等待回复的主动消息。
- quiet hours、每日上限和平台额度。
- 连续无响应、反感或暂停信号。
- 本次联系能否返还明确价值。

用户说“暂停、别提醒、停止、放弃”时，对应主动联系必须真实暂停。重新主动对话不等于重新授予主动联系权限。

主动联系授权必须绑定到具体 QQ 会话。Web 侧开启时只可默认使用已绑定的 QQ 单聊；不能因为“最近使用的是群聊”就把私人目标提醒发到群里。

## 6. 自动调整边界

Agent 可自动执行：

- 保存普通事实和反馈。
- 更新行动状态和推进证据。
- 缩小当前行动但不降低最终目标。
- 调整同一路径中的执行顺序、提示时机和表达方式。
- 生成或更新日志、复盘和下一行动。

Agent 必须请求确认：

- 改变用户最终目的或成功标准。
- 延长关键截止时间。
- 暂停、放弃或切换主目标。
- 开启主动联系。
- 执行有外部影响或不可逆的动作。

## 7. 体验准则

- 一次只推进一个关键点，最多问一个问题。
- 用户反馈不是打卡；每次反馈必须换回更低摩擦、更明确或更安全的下一步。
- 内部结构可以复杂，用户表达必须简单。
- 系统失败不能伪装成空数据或保存成功。
- “在线”“已发送”“已暂停”“已完成”必须有真实状态依据。
- AI 不以羞辱、恐惧、机械施压或人格标签推动行动。

## 8. 成功标准

一个用户连续 7 天只通过同一对话入口：

- 建立并确认一个目标。
- 每天收到或主动查询当前行动。
- 用短句反馈真实情况。
- 看到行动、路径或提示时机随反馈发生可解释变化。
- 不需要进入 Goals、Logs 或 Today 维护数据。
- 能随时询问 AI 当前如何理解目标以及为什么安排这一步。
- 说暂停后不再收到主动联系。

该标准可先通过确定性 API/运行时测试证明逻辑成立；真实 QQ 送达和真实用户长期价值仍需独立外部验收。
