# Page Positioning

设计理念：低熵极简主义。AI 负责复杂的目标管理方法，用户只需要看懂当前状态和下一步。

## Today

形式：四象限 + 下一步卡片 + 独立 Momentum 热力图小面板。

目的：告诉用户今天最该做什么。

页面边界：Today 不展示完整诊断、完成标准、复盘和计划细节。用户只需要一眼看到下一步要做什么，复杂判断由 Agent 和后台引擎处理。

充分必要条件：今日任务、优先级、当前最该做的任务、所属目标、任务状态、横向 GitHub 风格最近推进反馈。

## Goals

形式：页内胶囊切换，分为 OKR 与拆解甘特两个视角。

目的：让用户只读地看清长期目标、执行拆解、时间安排、验收点和风险。

页面边界：Goals 是只读总控台，不负责编辑、不负责 Check-in、不负责对话。OKR 视角负责把“大目标 -> 验收结果 -> 关键动作 -> 当前进度”的关系讲清楚；拆解甘特视角负责展示 WBS 工作项如何落到时间线上。

充分必要条件：Objective、KR/验收结果、WBS 工作项、Gantt 时间条、Milestone、Status、Today 列。

## Logs

形式：嵌套文件列表 + Markdown 编辑器，参考 `01. Daily/Year、Phase、Month、Weekly、Day` 的命名结构，但展示上按 Year -> Quarter -> Month -> Week -> Day 层级展开。

目的：让用户用最直接的文字方式，看清楚目标是如何从年度、季度、月度、周度推进到今天的。

页面边界：日志不是单纯日记，也不是额外报告系统。它是 Goal Mate 最终产出的 Markdown 文件，用户可以直接编辑，AI 也可以把 OKR、今日行动和复盘内容写入对应文件。

当前不做 MOC、Ideas、Knowledge。Goal Mate 服务行动推进，不做独立知识管理或灵感收纳。

充分必要条件：年/季/月/周/日文件层级、当前 MD 文件、可编辑文本、必要的 OKR 引用、今日行动、简单复盘区。

## Agent

形式：对话历史 + 专业工作台式聊天窗口。

目的：让用户通过微信/飞书式对话把信息交给机器人，并确认机器人准备执行的动作。

页面边界：Agent 不是审批控制台。它是一个充分了解 Goals、Logs、Today 的对话入口。确认动作只在聊天消息里临时出现，不常驻右侧面板。

充分必要条件：历史对话、当前聊天、输入框、可读取上下文提示、消息内确认动作。

## Settings

形式：左侧设置分类 + 右侧当前分类详情与具体配置控件。

目的：设置账号、目标系统、日志系统、Today、Agent、提醒、集成、数据与隐私。

页面边界：Settings 不是只管理 Agent 权限，也不能只是一堆开关。需要能配置路径、模板、数量、时间、模型、渠道、导出格式等实际参数。

充分必要条件：General、Goals、Logs、Today、Agent、Notifications、Integrations、Data & Privacy。
